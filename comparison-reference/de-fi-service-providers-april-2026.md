### 7. DeFi Insurance Protocols (new section)

47. **Nexus Mutual**  
   Core Mechanism: Mutual insurance model where members pool capital (NXM token) to provide cover against smart contract exploits, de-pegs, governance attacks, oracle failures, custodian hacks, and halted withdrawals; policyholders buy coverage and members vote on claims via the claims assessment process.  
   User-POV Safety & Security Facts: Multiple smart-contract audits conducted (newest to oldest listed in official resources); GitHub repositories contain contracts with public audit references; no single admin key control on core protocol; claims process is on-chain and member-voted; bug bounty program active.  
   Key Links: https://docs.nexusmutual.io/resources/audits-and-security/ | https://github.com/nexusmutual | https://github.com/NexusMutual/smart-contracts

48. **InsurAce**  
   Core Mechanism: Decentralised multi-chain insurance protocol providing risk protection services for smart contract vulnerabilities, custodian risk, stablecoin de-pegs, and other events across Ethereum, BSC, Solana, Polygon, Fantom, and additional chains.  
   User-POV Safety & Security Facts: Open-source contracts under InsurAce-Protocol organisation; multi-chain deployment with no single custody of funds; historical audits referenced in third-party reviews (limited public code coverage noted in older analyses).  
   Key Links: https://github.com/InsurAce-Protocol | https://www.insurace.io/ (protocol references in GitHub)

49. **Unslashed Finance**  
   Core Mechanism: Risk-specific capital pools where liquidity providers deposit ETH (or fractions) in exchange for capital tokens (CTKN); policyholders pay premiums for coverage (premium tokens PTKN) against smart contract failures, exchange hacks, oracle malfunctions, validator slashing, stablecoin de-pegs; claims challenged and resolved via Kleros decentralised court; capital invested via Enzyme Finance for yield.  
   User-POV Safety & Security Facts: Governance token USF (COMP-fork with checkpoints and GovernorAlpha); open-source governance contracts; separate capital pools per risk; no KYC required; claims use independent Kleros arbitration; 3 audits documented with open-source contracts.  
   Key Links: https://github.com/Unslashed/unslashed-governance | https://unslashed.finance/ (protocol mechanics in linked analyses referencing contracts)

50. **Tidal Finance**  
   Core Mechanism: Decentralised insurance marketplace (v2) where third-party underwriters/pool managers launch on-chain insurance policies with custom pricing and duration; separate collateral pools per policy; liquidity providers deposit to earn yield; policyholders purchase coverage; committees vote on payout proposals, manager changes, or committee membership.  
   User-POV Safety & Security Facts: Collateral separation between pools contains individual pool failures; fully decentralised underwriting by external parties; on-chain policy creation and claims voting; no single point of failure from centralised underwriting.  
   Key Links: https://docs.tidal.finance/ | https://github.com/tidalfinance | https://github.com/TidalFinance/tidal-contracts

51. **Bridge Mutual**  
   Core Mechanism: Decentralised coverage protocol offering protection for smart contract vulnerabilities, stablecoin de-pegs, and centralised exchange issues; liquidity providers contribute to pools that underwrite policies; protocol-owned yield generated by investing in third-party DeFi protocols; policy payouts and withdrawals handled on-chain.  
   User-POV Safety & Security Facts: Contracts audited by Consensys Diligence; open-source repositories with public SDK and v2 contracts; no single admin control on core coverage and payout functions; liquidity pools separate risks.  
   Key Links: https://github.com/Bridge-Mutual | https://github.com/Bridge-Mutual/bridgemutual-package | https://github.com/Bridge-Mutual/Bridgemutual-v2 | Consensys audit: https://consensys.net/diligence/audits/2021/03/bridge-mutual/

52. **Sherlock Protocol**  
   Core Mechanism: On-chain protocol providing smart contract exploit protection via proprietary security analysis; protocols pay fees for tailored coverage agreements; Sherlock repays up to $2M in the event of a missed vulnerability found in covered audits; integrates with DeFi users and protocol teams for coverage.  
   User-POV Safety & Security Facts: Open-source audit and coverage reports; auditors align incentives by repaying on missed vulns; public coverage agreements and reports; no custody of user funds; bug bounty and contest model for vulnerability discovery.  
   Key Links: https://github.com/sherlock-protocol/sherlock-docs | https://github.com/sherlock-protocol/sherlock-reports | https://github.com/sherlock-protocol

53. **Risk Harbor**  
   Core Mechanism: Parametric risk management marketplace using on-chain invariant detection for automated, transparent payouts; underwriters create pools with predetermined parameters; policyholders purchase coverage against smart contract risks, hacks, and attacks verifiable via on-chain metrics (no off-chain events).  
   User-POV Safety & Security Facts: Fully automated parametric triggers reduce dispute risk; open-source components referenced in DeFi risk tool lists; no KYC; pools isolate risks via on-chain metrics.  
   Key Links: https://www.riskharbor.com/ (referenced in GitHub risk tool lists) | https://github.com/defi-defense-dao/defi-risk-tools-list (protocol inclusion)
